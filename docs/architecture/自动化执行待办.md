# PRD + UI 自动化执行待办（执行级）

> 目标：用于一次触发后可直接逐项执行与勾选。

## 0. 环境检查

- [ ] 0.1 检查 Node.js 版本满足 `>= 22.22.0`。
- [ ] 0.2 检查 pnpm 版本满足 `>= 10.32.0`。
- [ ] 0.3 若版本不满足，停止执行并提示升级。

## 1. 输入准备

- [ ] 1.1 确认 `docs/prd/` 目录存在。
- [ ] 1.2 确认 `docs/ui/` 目录存在。
- [ ] 1.3 检查 PRD 文件编码与格式可读。
- [ ] 1.4 检查 UI 截图可访问、可解析。
- [ ] 1.5 校验 PRD 命名符合功能 `slug` 约定。
- [ ] 1.6 校验 UI 文件命名与 PRD 前缀可配对。
- [ ] 1.7 生成 PRD 文件名排序清单。
- [ ] 1.8 为每个 PRD 生成 UI 配对映射表。

## 2. 意图与范围确认

- [ ] 2.1 识别用户输入是否命中 PRD+UI+实现意图。
- [ ] 2.2 判定任务类型为“自动实现链路”。
- [ ] 2.3 写入任务唯一 ID（全流程追踪）。
- [ ] 2.4 输出本轮任务范围（本次处理哪些 PRD）。

## 3. OpenSpec 阶段（每个 PRD）

- [ ] 3.1 创建/选择 `change-id`（动词前缀 + kebab-case）。
- [ ] 3.2 执行 Proposal（逻辑命令 `openspec proposal`）。
- [ ] 3.3 生成 `proposal.md`（Why/What/Impact）。
- [ ] 3.4 生成 `tasks.md`（可执行清单）。
- [ ] 3.5 生成 `spec delta`（ADDED/MODIFIED/REMOVED）。
- [ ] 3.6 若复杂变更则生成 `design.md`。
- [ ] 3.7 执行 `openspec validate --strict`。
- [ ] 3.8 校验失败时修复后再次验证。
- [ ] 3.9 Proposal 通过后默认确认进入 Apply（除非用户明确拒绝）。
- [ ] 3.10 执行 Apply（逻辑命令 `openspec apply`）并按 `tasks.md` 逐项实施。

## 4. 设计分析阶段（每个 PRD）

- [ ] 4.1 对配对 UI 执行布局 Map 分析。
- [ ] 4.2 提取文字、图片、布局、层级四要素。
- [ ] 4.3 产出 UI 分析清单。
- [ ] 4.4 如 UI 缺失，记录“UI 待补”并继续。

## 5. 代码实现阶段（每个 PRD）

- [ ] 5.1 根据 PRD 语义拆分路由与组件边界。
- [ ] 5.2 创建/更新路由目录（Page/Loader/index.module.scss）。
- [ ] 5.3 创建页面级组件与通用组件。
- [ ] 5.4 完成接口类型定义（interfaces）。
- [ ] 5.5 完成接口封装（http）。
- [ ] 5.6 如需全局状态，创建 Pinia store。
- [ ] 5.7 样式全部使用主题变量与模块化样式。
- [ ] 5.8 接口未就绪时启用 mock 并标注替换点。
- [ ] 5.9 命中高风险操作时触发人工确认。

## 6. 质量门禁阶段（每个 PRD）

- [ ] 6.1 TypeScript 类型检查通过。
- [ ] 6.2 ESLint 检查通过。
- [ ] 6.3 关键路径单测通过。
- [ ] 6.4 关键页面冒烟测试通过。
- [ ] 6.5 E2E 测试通过。
- [ ] 6.6 构建（Vite build）通过。
- [ ] 6.7 若失败，自动修复并重试（最多 2 次）。
- [ ] 6.8 超过重试上限则标记该 PRD 失败并继续下一个。

## 7. UI 验收阶段（每个 PRD）

- [ ] 7.1 按截图优先原则执行 UI 对比验收。
- [ ] 7.2 记录 UI 问题清单（P0/P1/P2）。
- [ ] 7.3 修复阻断级 UI 问题并回归验证。
- [ ] 7.4 允许少量非阻断问题并明确留档。

## 8. 汇总与交付阶段

- [ ] 8.1 汇总每个 PRD 的通过/失败状态。
- [ ] 8.2 汇总待补 UI 与待替换 mock 清单。
- [ ] 8.3 汇总风险项与处置建议。
- [ ] 8.4 验收通过后执行 `openspec archive <change-id> --yes`。
- [ ] 8.5 归档后执行 `openspec validate --strict`。
- [ ] 8.6 生成执行日志记录。
- [ ] 8.7 输出“上线就绪结论”（不含部署）。
- [ ] 8.8 输出后续部署前检查建议。
